# ADR-0004: 엔트리 상세 페이지 — 향후 확장 결정 사항

**상태**: 보류 (향후 별도 스코프)
**날짜**: 2026-03-17
**관련**: ADR-0001 ~ 0003

## 개요

엔트리 상세 페이지 브레인스토밍 과정에서 논의되었으나, 현재 스코프에서 제외하고 향후 별도로 다루기로 한 결정들을 기록한다.

## 1. 단축 URL

- **배경**: 유니코드 slug를 사용하면 URL 공유 시 percent-encoding으로 길어질 수 있음
- **논의**: `dj-node.com/s/x7k2m` 같은 자체 단축 URL 또는 Vercel redirect 활용
- **결정**: 별도 스코프로 분리. 상세 페이지 완성 후 진행.

## 2. Title 변경 시 slug 동기화 정책

- **배경**: 사용자가 엔트리 title을 변경하면 slug도 바꿀지 결정 필요
- **옵션**:
    - slug도 변경 → 기존 공유 링크 깨짐
    - slug 유지 → title과 slug 불일치
    - slug 변경 + 기존 slug redirect → 가장 안전하지만 복잡
- **결정**: 구현 단계에서 결정

## 3. 대시보드 설정 객체 통합

- **배경**: 상세 페이지와 대시보드 디테일 뷰가 유사한 설정 구조를 가질 수 있음
- **현재**: 설정 객체를 분리하여 각각 관리
- **향후**: 두 설정 객체의 공통 인터페이스를 추출하여 통합. 한 곳 수정으로 양쪽 반영 가능하게.
- **결정**: 상세 페이지 설정 객체가 안정화된 후 통합 진행
